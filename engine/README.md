# Lute Tab Core

C++ typesetter for lute tablature. Processes plain text `.tab` files into PostScript, PDF, DVI, ASCII, or MIDI output.

## Architecture

Two-pass processing system (modelled after TeX):

- **Pass 1** (`src/input/`): Parse `.tab` files → internal representation
- **Pass 2** (`src/output/`): Internal format → output formats

## Building

```bash
make                    # Build the tab binary
make clean             # Remove build artifacts
./tab examples/simple.tab   # Test compilation
```

## Font Configuration

Fonts are loaded from the `fonts/` directory. Configure via:

1. `.env` file: `TABFONTS=/path/to/fonts`
2. Environment variable: `export TABFONTS=/path/to/fonts`
3. Build-time override: `make TLOC='"/custom/path"'`

The `fonts/` directory contains three artefact tiers from the same METAFONT sources (`*.mf`): `*.tfm` metrics and `*.pk` bitmaps consumed by the engine for PostScript output, plus a `woff2/` subdirectory with vector versions consumed by the desktop app's webview (CSS `@font-face`) and Skia-based screenshot rasteriser. The WOFF2 files are generated by `fonts/build-woff2-fonts.sh` and only need rebuilding when the `.mf` sources change. Body-text fonts (titles, lyrics) are not bundled — the engine emits PostScript names like `Helvetica`/`Times`/`NewCenturySchlbk` and trusts the consumer (Ghostscript or the webview) to supply them.

## Directory Structure

```
packages/core/
├── src/
│   ├── input/       # Pass 1: parsing and input handling
│   ├── output/      # Pass 2: format generators (PS/PDF/DVI/ASCII/MIDI)
│   ├── layout/      # Layout engine (beams, scoring, drawing)
│   ├── fonts/       # Font loading (TFM/PK)
│   ├── core/        # Utilities (args, buffer, debug)
│   └── main.cc      # Entry point
├── include/         # Header files
├── fonts/           # METAFONT sources + .tfm/.pk/.woff2 derivatives
└── examples/        # Sample .tab files
```

## Output Formats

- **PostScript** (default): `./tab file.tab` → `out.ps`
- **PDF**: `./tab -pdf file.tab`
- **DVI**: `./tab -dvi file.tab`
- **ASCII**: `./tab -ascii file.tab`
- **MIDI**: `./tab -midi file.tab`
