# Contributing to Tabbo

Thank you for your interest in contributing to Tabbo!

## Code of Conduct

Please be respectful and constructive in all interactions.

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test thoroughly with various .tab files
5. Commit with clear messages
6. Push to your fork
7. Open a Pull Request

## Development Setup

See the repository [README](../README.md) for setup instructions.

## Testing

Before submitting:
- Test with the included sample files (simple.tab, demo.tab, sample.tab)
- Ensure PostScript output generates correctly
- Verify no compilation warnings beyond existing ones

## Reporting Issues

Please include:
- Your OS and compiler version
- Steps to reproduce
- Sample .tab file if relevant
- Error messages or unexpected output

## Original Work

Please respect that Tab is Wayne Cripps' original work. Any contributions should:
- Maintain backward compatibility
- Preserve original copyright notices
- Follow the existing code style where possible
